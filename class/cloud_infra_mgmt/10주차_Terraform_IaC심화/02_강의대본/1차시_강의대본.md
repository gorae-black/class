## [슬라이드 05] 혼자 쓸 땐 몰랐던 문제들

"지난 주 배운 대로, 지금까지 State 파일은 전부 여러분의 VM 로컬 디스크에만 있었습니다. 혼자 쓸 땐
문제가 없어 보이지만, 이 방식은 세 가지 문제를 가지고 있습니다.

**협업 문제** — 여러 사람이 같은 인프라를 다루면, 각자의 State가 서로 달라 충돌이 날 수 있습니다.
**보안 문제** — State 안에 비밀번호 같은 민감정보가 평문으로 포함될 수 있습니다.
**실행 환경 문제** — 내 VM이 꺼지면, 아무도 그 인프라에 대해 apply를 실행할 수 없습니다.

**(질문 던지기)** 그럼 State를 어딘가 '공용 장소'에 두면 어떨까요? 바로 이 아이디어가 오늘 배울
**원격 백엔드(Remote Backend)**입니다."

## [슬라이드 07] Terraform Cloud가 대신 해주는 일

"Terraform Cloud는 크게 세 가지를 대신 해줍니다.

**State 저장** — 여러 사람이 공유하는 안전한 원격 저장소에 State를 보관합니다.
**원격 실행** — plan/apply를, 내 VM이 아니라 Terraform Cloud의 서버에서 실행합니다.
**실행 이력** — 누가 언제 무엇을 바꿨는지 'Runs' 탭에서 전부 기록하고 조회할 수 있습니다.

**(그림 설명)** 화면의 그림을 보시면, 내 VM은 코드를 작성하는 역할만 하고, 실제 State 저장과 실행은
Terraform Cloud가 맡습니다."

---

## [슬라이드 08] 원격 실행(Remote Execution)이란

"조금 더 자세히 설명드릴게요. `terraform apply`를 실행해도, 실제 plan/apply 연산은 **내 VM이
아니라 Terraform Cloud의 서버**에서 처리됩니다.

내 VM은 코드를 Terraform Cloud에 전달하는 역할만 하고, 실행이 끝나면 결과를 돌려받는 거예요.

**(협업 이점 재확인)** 이렇게 하면 여러 명이 같은 워크스페이스에 접근해도 항상 하나의 State만
존재하게 되어, 지난 시간에 걱정했던 충돌 문제가 사라집니다."

---

## [슬라이드 09] 오늘 꼭 기억해야 할 것

"자, 여기서 오늘 가장 중요한 포인트를 짚고 넘어가겠습니다.

**(핵심 경고)** 원격 실행은 Terraform Cloud의 **서버**에서 이뤄지기 때문에, 내 VM의 로컬 자원(예:
Docker)에는 접근할 수 없습니다.

그래서 10주차 실습에서는 9주차에 썼던 `docker` Provider 예제를 그대로 쓸 수 없습니다. 실제로
`docker_container`를 원격 실행하면 \"Cannot connect to the Docker daemon\" 오류가 납니다 — 원격
서버는 여러분의 VM Docker 데몬에 접속할 방법이 없으니까요.

그래서 다음 실습에서는 로컬 인프라에 의존하지 않는 리소스(`random_pet`)로 원격 실행의 원리만
확인할 예정입니다."

## 강의자 참고 — 예상 질문 & 답변

| 예상 질문 | 답변 가이드 |
|---|---|
| "Terraform Cloud는 유료인가요?" | 개인·소규모 팀용 무료 플랜이 있습니다. 이 과목 실습 범위에서는 무료 플랜으로 충분합니다. |
| "원격 실행이면 내 VM은 이제 필요 없나요?" | VM은 여전히 필요합니다. 코드를 작성하고 Git으로 관리하는 작업, 그리고 이후 주차의 Docker·K8s 실습은 계속 VM에서 진행됩니다. Terraform Cloud는 plan/apply '실행'만 대신해주는 것입니다. |
| "State를 원격에 두면 완전히 안전한가요?" | 로컬에 두는 것보다 훨씬 안전하지만, Terraform Cloud 계정 자체의 보안(비밀번호 관리, 접근 권한 설정)은 여전히 중요합니다. 이 과목에서는 기본적인 사용법까지만 다룹니다. |
| "Killercoda 링크가 언제 만들어졌는지에 따라 제한 시간이 달라지나요?" | 아니요. 60분 타이머는 링크 생성 시점이 아니라 각 학생이 시나리오를 '시작'한 순간부터 개별적으로 시작됩니다. 링크 자체는 만료되지 않으므로, 비동기로 늦게 접속하거나 여러 학생이 같은 링크를 써도 각자 독립된 환경과 타이머를 받습니다. |
