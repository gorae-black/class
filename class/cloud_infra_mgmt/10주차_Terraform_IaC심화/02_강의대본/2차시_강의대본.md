## [슬라이드 08] Workspace의 역할

"**워크스페이스**는 하나의 인프라 단위를 관리하는 작업 공간입니다. 그 안에 State, 변수, 실행
이력(Runs)이 함께 관리됩니다.

**(구조 설명)** 화면의 그림을 보시면, 조직(Organization) 아래에 워크스페이스가 있고, 그 워크스페이스
안에 State와 변수, 실행 이력이 들어있습니다. 코드(main.tf)를 어느 워크스페이스와 연결하느냐에 따라
State가 저장되는 곳이 정해집니다.

오늘은 `cloud-lab`이라는 이름으로 워크스페이스를 하나 만들 겁니다."

## [슬라이드 09] terraform login — CLI와 계정 연동

"이제 VM 터미널에서 Terraform Cloud 계정을 사용할 수 있도록 연동해야 합니다.

`terraform login` 명령을 실행하면 브라우저가 열리면서 토큰 인증이 진행됩니다. 발급된 토큰은 VM
로컬에 자동으로 저장되고, 이후 `terraform` 명령을 실행하면 자동으로 내 계정으로 인증됩니다.

**(보안 강조)** 이 토큰은 비밀번호와 같은 역할을 합니다. 다른 사람과 공유하거나, 코드에 그대로
남기지 않도록 주의하세요."

---

## [슬라이드 10] main.tf의 cloud 블록

"워크스페이스를 코드와 연결하려면 main.tf에 `cloud` 블록을 추가합니다.

```hcl
terraform {
  cloud {
    organization = "my-org"
    workspaces { name = "cloud-lab" }
  }
}
```

**(핵심 설명)** 이 블록이 있으면, `terraform init`을 실행할 때 State가 로컬이 아니라 지정한
워크스페이스(`organization`과 `workspaces.name`으로 지정한 곳)에 저장됩니다. `organization`
자리에는 여러분이 만든 실제 조직 이름을 넣어야 합니다."
