"GitHub는 코드를 저장하고 관리하는 서비스입니다. 이미 계정이 있으면 건너뛰어도 됩니다.

**(화면 공유해서 실제로 시연)** github.com에 접속해서 Sign up을 누르고, 이메일·비밀번호·사용자명을

## [슬라이드 08] app.py — 최소한의 웹 서버

"Flask는 파이썬으로 웹 서버를 아주 간단하게 만들 수 있게 해주는 라이브러리입니다.

오늘 만들 앱은 딱 다섯 줄입니다. `/`라는 주소로 접속하면 'Hello, Cloud!'라는 문자열 하나를 응답하는
게 전부예요.

**(부담 낮추기)** 이 코드를 외울 필요는 전혀 없습니다. 오늘 목표는 파이썬 문법을 익히는 게 아니라,
'내가 만든 앱을 Dockerfile로 이미지화하는 전체 흐름'을 경험하는 것입니다."

---

## [슬라이드 09] Dockerfile 작성

"지난 시간에 배운 명령어들을 이 앱에 그대로 적용해보겠습니다.

`FROM python:3.11-slim`으로 파이썬이 설치된 기반 이미지를 가져오고, `WORKDIR /app`으로 작업 폴더를
정합니다. `COPY app.py .`로 방금 만든 파일을 이미지 안에 넣고, `RUN pip install flask`로 Flask
라이브러리를 설치합니다. `EXPOSE 5000`으로 5000번 포트를 쓴다고 알려주고, 마지막 `CMD ["python",
"app.py"]`로 컨테이너가 시작될 때 이 앱을 실행하도록 합니다."

---

## [슬라이드 10] Killercoda — Flask 앱 빌드

"자, 직접 해보겠습니다.

`app.py`와 `Dockerfile`을 각각 작성하고, `docker build -t myflask:v1 .`으로 빌드하세요. 빌드가
끝나면 `docker run -d -p 5000:5000 myflask:v1`로 실행하고, `curl localhost:5000`을 쳐서 'Hello,
Cloud!'가 나오는지 확인하세요.

**(화면 공유해 실제로 시연)** 제가 먼저 전체 과정을 보여드릴게요."

---

## [슬라이드 11] VM 과제

"개인 VM에서 동일하게 `app.py`, `Dockerfile`을 작성하고 빌드·실행해서 curl 결과를 확인하세요.

그다음 이 두 파일을 아까 만든 GitHub 저장소(`cloud-lab`)에 `git add`, `git commit`, `git push`로
올려주세요. Git 명령어가 익숙하지 않은 분들을 위해 06_로컬VM_과제복습 폴더에 단계별 가이드를
올려두었습니다.

**(범위 명확화)** 오늘 필수 제출물은 여기까지입니다 — 로컬에서 빌드하고 실행해서 확인한 결과와 GitHub
링크만 제출하시면 됩니다. Docker Hub에 이미지를 올리는 건 다음 3차시의 선택 심화 과제이니, 오늘은
부담 갖지 않으셔도 됩니다."
