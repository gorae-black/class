## [슬라이드 05] 레이어는 순서대로 재사용된다

"Dockerfile의 각 줄은 하나의 '레이어'가 되어 저장됩니다. 그런데 다시 빌드할 때, **바뀌지 않은
앞부분의 레이어는 새로 안 만들고 그대로 재사용**합니다. 이걸 '캐시'라고 해요.

문제는, 중간에 바뀐 줄이 하나라도 있으면 **그 아래에 있는 레이어는 전부 다시 만들어야 한다**는
겁니다.

**(핵심 강조)** 그래서 Dockerfile을 쓸 때는, 자주 안 바뀌는 것(예: 라이브러리 설치)을 위쪽에,
자주 바뀌는 것(예: 내 코드 COPY)을 아래쪽에 배치하는 게 좋습니다."

---

## [슬라이드 06] 나쁜 순서 vs 좋은 순서

"표로 비교해보겠습니다. 만약 `COPY app.py .`를 `RUN pip install flask`보다 먼저 써버리면 — 코드
한 줄만 고쳐도 COPY 레이어가 바뀌었으니, 그 아래 있는 `pip install`까지 처음부터 다시 실행됩니다.
매번 느려요.

반대로 `RUN pip install flask`를 먼저 쓰고 `COPY app.py .`를 나중에 쓰면 — 코드만 고쳤을 때 설치
과정은 캐시를 그대로 재사용하고, COPY부터만 다시 하면 됩니다. 훨씬 빠르죠."

## [슬라이드 08] .dockerignore란?

".dockerignore는 Git을 써보신 분이라면 익숙할 .gitignore와 비슷합니다. 빌드할 때 이미지 안에
넣지 않을 파일이나 폴더를 지정하는 거예요.

예를 들어 `__pycache__/`, `.git/`, `*.log` 같은 건 굳이 이미지에 들어갈 필요가 없죠. 이런 파일들이
빠지면 이미지 용량도 줄고 빌드 속도도 빨라집니다.

**(중요성 강조)** 특히 `COPY . .`처럼 폴더 전체를 통째로 복사할 때, .dockerignore가 없으면 불필요한
파일까지 전부 이미지에 들어가버리니 꼭 챙겨야 합니다."

## [슬라이드 09] Killercoda — 최적화 전후 비교

"자, 직접 체감해보겠습니다.

먼저 `RUN pip install`을 `COPY app.py .` 아래에 두는 '나쁜' Dockerfile로 빌드하고 시간을 기록하세요.
그다음 `app.py` 내용을 한 줄만 수정하고 다시 빌드해서, 또 처음부터 오래 걸리는 걸 확인하세요.

이번엔 `RUN`을 `COPY`보다 위로 옮긴 '좋은' Dockerfile로 바꾸고, 다시 `app.py`를 수정한 뒤 빌드해보세요.
캐시 덕분에 훨씬 빨라진 게 느껴지실 거예요.

**(화면 공유해 실제로 시연)** 제가 먼저 두 가지 경우를 비교해서 보여드릴게요."

## [슬라이드 10] (선택 심화) Docker Hub push

"여기부터는 선택사항입니다. 완성한 이미지를 Docker Hub에 올려서 다른 사람과 공유해보고 싶은 분들은
해보세요.

`docker login`으로 Docker ID와 비밀번호를 입력해 로그인하고, `docker build -t {여러분의dockerhub_id}
/myflask:v1 .`처럼 본인 Docker ID를 이미지 이름 앞에 붙여서 다시 빌드합니다. 그다음 `docker push
{dockerhub_id}/myflask:v1`로 올리면 끝입니다. Docker Hub 웹사이트에 로그인해서 본인 저장소에 이미지가
올라갔는지 확인해보세요.

**(부담 낮추기)** 오늘 필수 제출물은 레이어 캐싱 비교 실습까지입니다. Docker Hub push는 여유가 되는
분만 해보시면 됩니다."

## 강의자 참고 — 예상 질문 & 답변

| 예상 질문 | 답변 가이드 |
|---|---|
| "docker push가 권한 오류로 안 돼요" | 이미지 이름이 본인 Docker ID로 시작하는지 확인하게 하세요(예: kimcloud/myflask). 다른 사람 이름으로는 push할 권한이 없습니다. |
| "캐시가 항상 좋은 건가요?" | 대부분 좋지만, 가끔 예전 캐시 때문에 최신 코드가 안 반영되는 것처럼 보일 때가 있습니다. 그럴 땐 docker build --no-cache로 캐시를 무시하고 처음부터 다시 빌드할 수 있다고 알려주세요(참고 정보로만 언급). |
| "이미지 용량을 얼마나 줄일 수 있나요?" | 오늘 배운 정도로는 큰 차이는 아니지만, 실무에서는 alpine 같은 경량 베이스 이미지 사용, 멀티스테이지 빌드 등을 함께 쓰면 수십 배까지 줄이기도 합니다. 이 과목에서는 다루지 않는 심화 주제이니 관심 있으면 스스로 찾아보도록 안내하세요. |
