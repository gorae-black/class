- 정식 제출 대상: **3차시**만 필수 제출입니다. 1·2차시는 VM 환경이 정상 동작하는지 스스로 확인하는
  용도로, 캡처를 남겨두면 좋지만 별도 제출은 필수가 아닙니다.

## 2. 실습 체크리스트

### 1차시 — 클라우드 패러다임·가상화 (자가 확인, 제출 선택)
- [ ] (Killercoda) `whoami`, `uname -a`, `nproc`, `free -h` 실행
- [ ] (VM) `docker --version`, `git --version` 실행 → 정상 출력 확인

### 2차시 — 컨테이너·Docker 아키텍처 (자가 확인, 제출 선택)
- [ ] (VM) `docker run hello-world` 실행
- [ ] (VM) `docker images`, `docker ps -a` 실행 → Killercoda 결과와 비교

### 3차시 — Docker 데몬·nginx 실습 (★ 필수 제출)
- [ ] (VM) `docker run -d --name web nginx` 실행
- [ ] (VM) `docker ps` 실행 결과 — 캡처 첨부
- [ ] (VM) `docker inspect web | grep IPAddress` 실행 결과 — 캡처 첨부
- [ ] (선택) 위 결과를 GitHub 저장소에 정리해 링크로 제출해도 무방

## 3. 자가 점검

- [ ] 3차시 캡처에 명령어와 결과가 함께 보인다 (터미널 프롬프트 포함)
- [ ] Killercoda에서 나온 컨테이너 IP와 VM에서 나온 IP가 다른 이유를 설명할 수 있다
