## 2. 실습 체크리스트

### 1차시 — ConfigMap과 Secret (★ 필수)
- [ ] `kubectl create configmap`으로 설정값 생성
- [ ] `kubectl create secret generic`으로 비밀값 생성
- [ ] `kubectl get configmap`, `kubectl get secret` 결과 — 캡처 첨부

### 2차시 — PV/PVC 실습 (★ 필수)
- [ ] `pv.yaml`(hostPath: `/data/mysql`, storage: `1Gi`), `pvc.yaml` 작성 및 적용
- [ ] `kubectl get pv`, `kubectl get pvc` 결과(STATUS: Bound 확인) — 캡처 첨부
- [ ] PV/PVC를 삭제하지 않고 유지했음

### 3차시 — MySQL 데이터 영속성 검증 (★ 필수, 6주차 종합)
- [ ] PVC를 마운트한 MySQL Pod 배포, 테이블 생성·데이터 삽입
- [ ] Pod 삭제 **전** SELECT 결과 — 캡처 첨부
- [ ] `kubectl delete pod` 실행 후, 새 Pod에서 SELECT **재확인** 결과 — 캡처 첨부 (10~20초 대기 필요)

## 3. 자가 점검

- [ ] Secret의 Base64 인코딩이 암호화가 아니라는 것을 설명할 수 있다
- [ ] PV와 PVC의 역할 차이를 설명할 수 있다
- [ ] 삭제 전/후 SELECT 결과가 동일함을 캡처로 증명했다
