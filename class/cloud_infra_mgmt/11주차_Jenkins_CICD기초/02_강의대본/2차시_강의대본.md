## [슬라이드 07] Jenkins docker-compose.yml (검증 완료)

"화면의 설정을 하나씩 볼게요.

```yaml
services:
  jenkins:
    image: jenkins/jenkins:lts
    ports: ["8080:8080", "50000:50000"]
    environment: ["JAVA_OPTS=-Xmx512m"]
    volumes:
      - jenkins_home:/var/jenkins_home
      - /var/run/docker.sock:/var/run/docker.sock
```

`image`는 Jenkins의 공식 LTS(장기지원) 버전을 씁니다. `ports`는 8080(웹 UI)과 50000(에이전트 통신용)
포트를 엽니다. `volumes`에 두 가지가 마운트되어 있는데, 이 두 개가 오늘의 핵심입니다."

---

## [슬라이드 08] docker.sock을 마운트하는 이유

"먼저 `/var/run/docker.sock`부터 설명드릴게요.

**(개념 설명)** 이건 호스트의 Docker 데몬과 통신하는 '창구' 역할을 하는 소켓 파일입니다. 이걸
Jenkins 컨테이너 안에 그대로 마운트하면, Jenkins가 컨테이너 안에서도 **호스트의 Docker를 직접
제어**할 수 있게 됩니다.

**(그림 설명)** 화면의 그림을 보시면, Jenkins 컨테이너가 이 소켓을 통해 호스트(VM)의 Docker
데몬과 직접 연결됩니다.

**(13주차 예고)** 13주차에 Jenkins가 `docker build`, `docker push`를 실행할 때 바로 이 구조를
사용합니다. 지금은 '이런 연결 통로가 있구나' 정도로 기억해두시면 됩니다."

---

## [슬라이드 09] JAVA_OPTS로 메모리 제한

"두 번째로 `environment`의 `JAVA_OPTS=-Xmx512m`을 볼게요.

Jenkins는 자바로 만들어진 프로그램인데, 이 옵션으로 힙 메모리를 512MB로 제한해둡니다.

**(이유 설명)** 왜 이렇게 제한할까요? 13~14주차에 minikube와 Jenkins를 **동시에** 띄워야 하는데,
우리 VM은 RAM이 4GB로 제한되어 있습니다. Jenkins가 메모리를 너무 많이 쓰면 minikube가 불안정해질
수 있어서, 미리 여유 공간을 확보해두는 거예요.

**(팁 공유)** 14주차 부하테스트 직전에는 `docker stop jenkins`로 Jenkins를 잠시 꺼두는 것도 좋은
방법입니다."

## [슬라이드 10] Killercoda — Compose 문법 빈칸 채우기

"자, 직접 해보겠습니다. 뼈대만 있는 docker-compose.yml에서 `image`, `ports`, `volumes` 빈칸을

완성했으면 `docker compose up -d`로 실행하고, `docker compose logs`로 컨테이너 상태를 확인하세요.

**(이해도 확인)** 완성한 설정이 왜 그렇게 구성되는지 스스로 한 번 설명해보세요.

## [슬라이드 11] VM 안내 — Jenkins 설치 실습

"개인 VM에서 오늘 배운 docker-compose.yml을 그대로 작성하고 `docker compose up -d`를 실행하세요.

`docker compose logs jenkins`로 초기 관리자 비밀번호를 확인하고, 브라우저에서 `localhost:8080`에
접속해서 그 비밀번호를 입력하는 로그인 화면까지 캡처하세요.

**(범위 명확화)** 오늘은 여기까지가 필수 제출물입니다. 플러그인 설치나 관리자 계정 생성은 다음
3차시에 이어서 진행하니, 오늘은 초기 비밀번호 입력 화면까지만 진행하시면 됩니다."

## [슬라이드 12] 오늘의 정리

"오늘 배운 내용, 두 가지로 정리하겠습니다.

첫째, **Named Volume(jenkins_home)으로 컨테이너를 껐다 켜도 Jenkins 설정이 유지되게 한다.**

## 강의자 참고 — 예상 질문 & 답변

| 예상 질문 | 답변 가이드 |
|---|---|
| "docker compose logs에 비밀번호가 안 보여요" | Jenkins가 완전히 시작되기까지 몇 초~1분 정도 걸릴 수 있습니다. 잠시 기다린 후 다시 `docker compose logs jenkins`를 실행해보라고 안내하세요. 로그가 너무 길면 `docker compose logs jenkins | grep -A 3 password`로 찾을 수도 있습니다. |
| "docker.sock을 마운트하면 보안에 문제 없나요?" | 실무에서는 이 방식이 컨테이너에 호스트 수준의 권한을 준다는 점에서 보안 논의가 있는 주제입니다. 다만 이 과목은 개인 실습 VM 환경이라 별도 보안 대책 없이 바로 사용합니다. 실무에서는 더 안전한 대안(Docker-in-Docker, 별도 빌드 에이전트 등)을 검토합니다. |
| "포트 50000은 왜 필요한가요?" | Jenkins에 별도의 빌드 에이전트(다른 서버)를 연결할 때 쓰는 포트입니다. 이 과목에서는 별도 에이전트를 쓰지 않지만, 기본 설정에 포함되어 있어 그대로 열어둡니다. |
