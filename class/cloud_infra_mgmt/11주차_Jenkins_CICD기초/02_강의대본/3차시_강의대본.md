## [슬라이드 08] Freestyle Job의 기본 구조

"Job을 만드는 가장 간단한 방법이 **Freestyle Job**입니다.

**New Item**을 클릭하고 **Freestyle project**를 선택한 뒤 Job 이름을 입력합니다. 그다음 **Build
Steps**에 'Execute shell'을 추가하고, 실행할 명령어를 입력합니다.

저장한 뒤 **Build Now**를 클릭하면, Build 이력에 실행 결과(**Console Output**)가 남습니다.

**(그림 설명)** 화면의 그림처럼 'Job 생성 → Build Steps 설정 → Build Now → Console Output 확인'
이 네 단계가 Jenkins Job의 가장 기본적인 흐름입니다."

---

## [슬라이드 09] 오늘 만들 Job — hello-jenkins

"오늘 실습에서 만들 Job은 이렇습니다.

Job 이름은 `hello-jenkins`(Freestyle project)로 만들고, Build Steps의 Execute shell에
`echo \"Hello, Jenkins! Build number: $BUILD_NUMBER\"`를 입력합니다.

**(변수 설명)** `$BUILD_NUMBER`는 Jenkins가 자동으로 채워주는 환경변수로, 이 Job이 몇 번째로
실행됐는지를 나타냅니다.

Build Now를 실행하면, Console Output에서 echo 결과와 빌드 번호가 함께 출력되는 걸 볼 수 있습니다."

## [슬라이드 10] Killercoda — Job 생성 실습

"자, 직접 해보겠습니다. 임시 Jenkins 컨테이너에서 Freestyle Job을 처음부터 직접 만들어보세요.

Execute shell에 echo 명령을 3단계로 작성해보세요 — 예를 들어 '시작 로그' → '작업 로그' → '완료
로그' 순서로요. Build Now를 실행한 뒤 Console Output에서 이 로그들이 순서대로 찍히는지 확인하세요.

**(화면 공유해 실제로 시연)** 제가 먼저 보여드릴게요."

## 강의자 참고 — 예상 질문 & 답변

| 예상 질문 | 답변 가이드 |
|---|---|
| "Install suggested plugins 말고 직접 선택해서 설치해도 되나요?" | 가능하지만, 이 과목에서 필요한 플러그인(Git, Pipeline 등)이 suggested plugins에 대부분 포함되어 있어 자동 설치를 권장합니다. 나중에 필요한 플러그인이 빠졌다면 Jenkins 관리 화면에서 추가로 설치할 수 있습니다. |
| "Build Now를 눌렀는데 아무 반응이 없어요" | Jenkins 대시보드 왼쪽 하단이나 Build History에서 진행 중인 빌드(회전하는 원 아이콘)를 확인하라고 안내하세요. 빌드가 끝나면 파란 원(성공) 또는 빨간 원(실패)으로 표시됩니다. |
| "Freestyle Job과 다음 주에 배울 Pipeline은 뭐가 다른가요?" | Freestyle Job은 화면에서 클릭으로 설정하는 방식이고, Pipeline(Jenkinsfile)은 그 설정을 코드로 작성하는 방식입니다. 코드로 작성하면 Git으로 버전 관리를 할 수 있고, 복잡한 단계(Stage)를 구성하기도 더 유리해서 실무에서는 Pipeline 방식을 더 많이 씁니다. |
| "Killercoda 링크가 언제 만들어졌는지에 따라 제한 시간이 달라지나요?" | 아니요. 60분 타이머는 링크 생성 시점이 아니라 각 학생이 시나리오를 '시작'한 순간부터 개별적으로 시작됩니다. 링크 자체는 만료되지 않으므로, 비동기로 늦게 접속하거나 여러 학생이 같은 링크를 써도 각자 독립된 환경과 타이머를 받습니다. |
