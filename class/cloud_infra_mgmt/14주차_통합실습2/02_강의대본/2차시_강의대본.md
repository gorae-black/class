## [슬라이드 05] ab 명령어 기본 문법

"**ApacheBench**, 줄여서 `ab`는 터미널에서 간단하게 부하를 흉내낼 수 있는 도구입니다.

```
ab -n 10000 -c 100 http://{chatbot-url}/
```

`-n 10000`은 총 몇 번 요청을 보낼지(요청 총량)를 뜻하고, `-c 100`은 동시에 몇 개의 연결로 요청을
보낼지(동시성)를 뜻합니다. 실행하면 초당 처리 요청 수, 응답 시간 같은 통계가 출력됩니다.

**(비유)** 마치 순간적으로 수백 명이 동시에 우리 챗봇에 접속하는 상황을 인위적으로 만드는
거예요."

## [슬라이드 07] 부하를 줘도 CPU가 0%에 머무는 이유

"**(핵심 경고)** 만약 챗봇 앱이 요청당 실제 연산이 거의 없다면 — 예를 들어 단순히 미리 정해둔
문자열을 반환하기만 한다면 — `ab`로 아무리 부하를 줘도 CPU 사용률이 **0%에 머무를 수 있습니다.**

이건 여러분이 뭔가 잘못해서가 아니라, 원래 그렇게 동작하는 겁니다. 요청을 처리하는 데 CPU 연산이
거의 필요 없으니, 요청이 아무리 많이 와도 CPU가 바빠지지 않는 거예요.

**(추가 주의사항)** 그리고 우리 VM은 2 vCPU라서, `ab` 프로세스 자체가 오히려 더 큰 부담이 되기도
합니다.

**(해결책 제시)** 그래서 두 가지를 함께 적용해야 합니다. 첫째는 지난 시간에 이미 한
resource request를 낮게 설정하는 것, 둘째는 오늘 새로 할 챗봇 응답 로직에 약간의 연산을 추가하는
것입니다."

## [슬라이드 08] 챗봇 응답 로직에 연산 추가하기

"구체적으로는, `app.py`의 응답 함수 안에 문자열 처리를 반복하는 코드를 몇 줄 추가합니다. 예를 들어
특정 연산을 N번 반복하는 식이에요.

**(효과 설명)** 이렇게 하면 요청 하나당 실제로 CPU를 사용하는 시간이 생겨서, 부하가 몰릴 때
사용률이 눈에 띄게 오릅니다.

**(파이프라인 재활용 강조)** 코드를 수정한 뒤에는 어떻게 배포하죠? 맞습니다 — 12~13주차에 배운
파이프라인 그대로, `git push`만 하면 자동으로 새 버전이 배포됩니다. 오늘은 이 파이프라인을 실전에서
다시 한번 활용해보는 거예요."

## [슬라이드 09] VM 실습 — 부하 테스트 준비

"자, 직접 해보겠습니다.

먼저 `app.py`에 연산을 추가하고 `git push`하세요. 파이프라인이 자동으로 배포할 때까지 기다렸다가,
새 버전이 반영됐는지 확인하세요.

그다음 다른 터미널 창 두 개를 열어서, 하나에는 `kubectl get pods -w`, 다른 하나에는 `kubectl get
hpa -w`를 띄워두세요. 실시간으로 관찰할 준비를 하는 겁니다.

마지막으로 `ab -n 10000 -c 100 http://$(minikube service chatbot --url)/`를 실행하세요."

## [슬라이드 10] VM 실습 — 스케일업 관찰

"부하를 주는 동안 `kubectl get hpa -w` 화면을 지켜보세요. TARGET 항목의 CPU %가 점점 올라가는 걸
볼 수 있을 거예요.

목표치(50%)를 넘으면, REPLICAS가 1 → 2 → 3처럼 늘어나는 순간이 옵니다. 이 순간을 놓치지 말고
캡처하세요.

동시에 `kubectl get pods -w` 화면에서도 새로운 chatbot Pod가 추가로 생성되는 걸 확인할 수 있습니다.

**(감동 포인트 강조)** 이 순간이 이 과목에서 가장 극적인 순간 중 하나입니다. 여러분이 코드 한 줄
건드리지 않았는데, K8s가 스스로 판단해서 서버를 늘리는 걸 직접 보시는 거예요."

## 강의자 참고 — 예상 질문 & 답변

| 예상 질문 | 답변 가이드 |
|---|---|
| "연산을 추가했는데도 CPU가 안 올라요" | resource request가 실제로 적용됐는지(`kubectl describe deployment chatbot`), 추가한 연산이 충분히 무거운지(반복 횟수를 늘려보기), 새 이미지가 실제로 배포됐는지(`kubectl get pods`의 이미지 태그 확인) 순서대로 점검하도록 안내하세요. |
| "ab 실행 중 VM이 너무 느려져요" | 2 vCPU 환경에서는 `-c`(동시성) 값을 100에서 30~50 정도로 낮춰서 실행해보라고 안내하세요. 목표는 극한의 부하가 아니라 '스케일업이 관찰될 정도'의 부하입니다. |
| "REPLICAS가 max(5)까지 다 늘어나야 하나요?" | 아닙니다. 목표치를 넘는 순간 1개 이상 늘어나는 것만 확인되면 충분합니다. 부하 강도에 따라 2개에서 멈출 수도, 5개까지 늘어날 수도 있습니다. |
