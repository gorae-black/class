## [슬라이드 05] 컨테이너는 상태를 가진다

"컨테이너는 만들어지고(Created), 실행되고(Running), 멈추고(Stopped), 지워지는(Removed) 네 가지 상태를
거칩니다.

`docker create`는 컨테이너를 '만들기만' 하고 실행은 안 합니다. `docker start`가 그걸 실제로 실행시켜요.
그런데 우리가 지금까지 써온 `docker run`은 사실 이 둘을 합친 거예요 — create + start를 한 번에 하는
명령어입니다.

`docker stop`은 실행 중인 컨테이너를 멈춥니다. **중요한 건 stop을 해도 컨테이너 자체와 그 안의 파일은
사라지지 않는다는 것**입니다. 마치 컴퓨터를 끈 것과 비슷해요. 완전히 지우려면 `docker rm`을 써야 합니다."

## [슬라이드 08] logs와 exec, 언제 쓰나

"`docker logs {이름}`은 그 컨테이너가 지금까지 출력한 로그를 보여줍니다. `-f` 옵션을 붙이면 로그를
실시간으로 계속 지켜볼 수 있어요 (follow).

`docker exec -it {이름} bash`는 아예 그 컨테이너 '안으로' 직접 들어가는 명령어입니다. 마치 그 컨테이너
안에 있는 리눅스에 SSH로 접속한 것처럼, 컨테이너 내부에서 명령어를 자유롭게 칠 수 있어요. 나올 때는
`exit`을 치면 됩니다.

**(비유로 정리)** logs는 '창문 너머로 구경하기'고, exec는 '문 열고 직접 들어가기'라고 생각하시면 됩니다."

## [슬라이드 09] Killercoda — 생명주기 실습

"자, 직접 해보겠습니다.

`docker run -d --name web2 nginx`로 컨테이너를 하나 새로 실행하고, `docker stop web2` 한 다음
`docker ps -a`로 상태가 'Exited'로 바뀐 걸 확인하세요. 그다음 `docker start web2`로 다시 켜고 'Up'
상태인지 확인합니다.

마지막으로 `docker logs web2`로 로그를 보고, `docker exec -it web2 bash`로 컨테이너 안에 들어갔다가
`exit`으로 나와보세요.

**(화면 공유해 실제로 시연)** 제가 먼저 순서대로 보여드릴게요."

## 강의자 참고 — 예상 질문 & 답변

| 예상 질문 | 답변 가이드 |
|---|---|
| "stop과 rm -f 차이가 뭔가요?" | stop은 컨테이너를 멈추기만 하고 나중에 start로 다시 켤 수 있습니다. rm -f는 실행 중이어도 강제로 완전히 삭제해서 되돌릴 수 없습니다. |
| "exec로 들어가서 파일을 만들면 어떻게 되나요?" | 컨테이너 안에 저장됩니다. 하지만 오늘 배운 대로, 컨테이너를 rm으로 지우면 그 파일도 함께 사라집니다. 이 문제는 다음 시간 볼륨에서 해결합니다. |
| "docker ps에는 안 보이는데 docker ps -a에는 보이는 이유는?" | docker ps는 기본적으로 '실행 중(Running)'인 컨테이너만 보여줍니다. -a를 붙여야 Stopped 상태까지 포함한 전체 목록이 나옵니다. |
