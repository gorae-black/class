## Part 3. YAML 오류 수정 (30점, 문항당 15점)

### 15번 (15점)
- **오류 위치**: `containers:` 아래 `images: nginx:1.25`
- **원인**: K8s Pod spec의 컨테이너 필드명은 `image`(단수)이며, `images`는 존재하지 않는 필드라
  인식되지 않는다(또는 스키마 검증 오류/`unknown field`로 apply 실패).
- **올바른 값**: `image: nginx:1.25`
- **채점**: 오류 위치 지목 7점 + 원인 설명 4점 + 올바른 값 4점. "필드명 오타"라고만 써도 원인 설명은
  인정하되 "왜 문제인지"(필드를 인식 못해 컨테이너 이미지가 지정되지 않음/apply 실패)까지 쓰면 만점

### 16번 (15점)
- **오류 위치**: PVC의 `resources.requests.storage: 5Gi` (PV의 `capacity.storage: 1Gi`와 불일치)
- **원인**: PVC가 요청하는 용량(5Gi)이 PV가 제공하는 용량(1Gi)보다 크다. PVC는 자신의 요청 조건을
  만족하는 PV를 찾아 Bound되는데, 조건을 만족하는 PV가 없으므로 PVC가 계속 **Pending** 상태에
  머무른다.
- **올바른 값**: PVC의 `storage`를 `1Gi` 이하로 낮추거나, PV의 `storage`를 `5Gi` 이상으로 올려야 함
  (둘 중 하나를 맞게 쓰면 인정)
- **채점**: 오류 위치 지목 7점 + "PVC 요청이 PV 제공량보다 커서 Bound되지 못하고 Pending 상태가
  된다"는 원인 설명 4점 + 올바른 수정 방향 4점
