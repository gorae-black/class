# 8주차 1차시 — 1~3주 복습 퀴즈 (OX·단답형 10문항)

- 범위: 1주차(클라우드·가상화) ~ 3주차(Dockerfile·이미지 빌드·배포)
- 형식: OX 6문항 + 단답형 4문항
- 용도: LMS 자동채점 퀴즈. 수업 중 함께 풀고 바로 정답·해설을 확인합니다.

---

## OX 문제 (1~6번)

**1.** 클라우드 컴퓨팅의 종량제 과금은 사용자가 미리 정해진 고정 요금을 매달 내는 방식이다. ( )

**2.** 가상화에서 VM은 하이퍼바이저 위에서 각자 완전한 Guest OS를 가진다. ( )

**3.** 컨테이너는 VM과 달리 호스트와 OS 커널을 공유하기 때문에 VM보다 가볍고 빠르게 실행된다. ( )

**4.** Docker Image는 실행 중인 인스턴스이고, Container는 그 설계도이다. ( )

**5.** Bind Mount로 연결한 컨테이너를 삭제해도, 호스트(VM)에 저장된 데이터는 그대로 남아있다. ( )

**6.** Dockerfile에서 `RUN`은 컨테이너가 시작될 때마다 실행되는 명령이다. ( )

## 단답형 문제 (7~10번)

**7.** Docker에서 `docker ps` 명령에 실행 중이 아닌 컨테이너까지 모두 포함해서 보고 싶을 때 추가하는
옵션은 무엇인가요?

**8.** Named Volume과 Bind Mount의 차이를 한 문장으로 설명하세요.

**9.** 이미지 빌드 속도를 높이기 위해, Dockerfile에서 "자주 안 바뀌는 명령"과 "자주 바뀌는 명령"을
각각 위/아래 중 어디에 배치해야 하는지 쓰세요.

**10.** Docker Hub처럼 이미지를 저장하고 공유할 수 있는 저장소를 K8s/Docker 생태계에서 부르는 용어는
무엇인가요?

## 정답 및 해설

| 번호 | 정답 | 해설 |
|---|---|---|
| 1 | X | 종량제 과금은 **쓴 만큼만** 지불하는 방식이다. 고정 요금이 아니다. |
| 2 | O | VM은 하이퍼바이저가 물리 자원을 배분받아, 각자 독립된 Guest OS를 가진다. |
| 3 | O | 컨테이너는 OS 커널을 공유하므로 VM보다 가볍고 시작이 빠르다. |
| 4 | X | 반대다. Image가 설계도(정적), Container가 실행 중인 인스턴스(동적)다. |
| 5 | O | Bind Mount는 호스트 디렉토리를 그대로 연결하므로, 컨테이너가 삭제돼도 호스트에 데이터가 남는다. |
| 6 | X | `RUN`은 **빌드 시점**에 한 번 실행된다. 컨테이너 시작 시 실행되는 건 `CMD`다. |
| 7 | `-a` (`docker ps -a`) | `-a` 옵션을 붙이면 종료된 컨테이너까지 모두 표시된다. |
| 8 | (예시) Bind Mount는 호스트의 특정 경로를 직접 지정해 연결하고, Named Volume은 Docker가 관리하는 전용 저장공간을 연결한다. | 핵심은 "누가 저장 위치를 관리하는가"의 차이 — 채점 시 이 취지가 담기면 정답 처리 |
| 9 | 자주 안 바뀌는 명령은 **위**, 자주 바뀌는 명령은 **아래** | 위쪽 레이어부터 캐시가 재사용되므로, 변경이 적은 것을 위에 둬야 캐시 적중률이 높아진다. |
| 10 | 레지스트리(Registry) | Docker Hub는 대표적인 공개 레지스트리다. |
