## 문제

```hcl

  name  = "tf-nginx"

}
```

**3.** Terraform이 "지금 무엇을 만들었는지" 기억하는 파일의 이름은 `terraform.______`이다.

**4.** Terraform Cloud와 CLI를 연동할 때, 브라우저 토큰 인증을 진행하는 명령어는 `terraform
______`이다.

**5.** 아래는 워크스페이스와 연결하는 cloud 블록입니다. 빈칸을 채우세요.

```hcl

  cloud {
    organization = "my-org"

  }

```

```groovy
pipeline {
  agent any

```

**8.** Docker Hub 비밀번호처럼 민감한 정보를 Jenkinsfile 코드에 그대로 노출하지 않고 안전하게
사용하게 해주는 스텝은 `______Credentials`이다.

**9.** 학생 VM처럼 NAT 뒤에 있어 GitHub이 직접 접속할 수 없는 환경에서, Jenkins가 스스로 GitHub에

**10.** Job 설정의 Build Triggers에서 Poll SCM을 매 1분마다 실행하려면 Schedule란에 cron 형식으로
`______`을 입력한다.

## 정답 및 해설

| 번호 | 정답 | 해설 |
|---|---|---|
| 1 | `init` | `terraform init`은 코드에 명시된 Provider를 다운로드하고 작업 폴더를 초기화한다. |
| 2 | `resource` | HCL의 두 가지 핵심 블록은 `provider`(무엇을 다룰지)와 `resource`(무엇을 만들지)다. |
| 3 | `tfstate` | `terraform.tfstate` 파일에 현재 인프라 상태가 기록된다. |
| 4 | `login` | `terraform login`으로 브라우저에서 토큰 인증을 진행해 CLI와 계정을 연동한다. |
| 5 | `workspaces` | `workspaces { name = "..." }`로 어느 워크스페이스에 State를 저장할지 지정한다. |
| 6 | `Docker daemon` | 원격 실행 서버는 학생 로컬 Docker 데몬에 접근할 수 없어 이 오류가 발생한다. |
| 7 | `stages` | `stages` 블록 안에 여러 `stage`를 순서대로 나열한다. |
| 8 | `with` (`withCredentials`) | `withCredentials([...]) { ... }`로 등록된 Credential을 안전하게 코드 안에서 사용한다. |
| 9 | `Poll SCM` | Jenkins가 정해진 주기마다 스스로 저장소 변경 여부를 확인하는 방식이다. |
| 10 | `* * * * *` | cron 형식에서 다섯 개의 별은 "매 분마다"를 의미한다. |
