# 15주차 — 기말고사 시험범위 정리

- 범위: 9주차 ~ 14주차 (Terraform IaC + Jenkins CI/CD + 대통합 실습)
- 시험 방식: LMS 온라인 시험 / 전체 파이프라인 아키텍처 분석·설계 서술형 + 코드 트러블슈팅 (3차시)
- 1차시(9~12주 복습)와 2차시(13~14주 통합실습 체크포인트)는 이 문서를 기준으로 진행합니다.
- 참고: 1~8주차(Docker·K8s) 범위는 중간고사에서 이미 평가했으므로 기말고사에는 포함되지 않습니다.
  다만 13~14주차 실습은 그 위에 세워진 것이므로, 필요하면 `08주차_중간고사/01_시험범위정리`도 함께
  참고하세요.

## 1차시 복습 범위 — 9~12주차 (Terraform + Jenkins)

### 9주차 — Terraform 입문과 HCL 문법
| 개념 | 핵심 요약 |
|---|---|
| IaC(Infrastructure as Code) | 인프라를 코드로 선언하고, 그 코드로 생성·변경·삭제하는 방식 |
| Terraform 아키텍처 | HCL 코드 → Terraform CLI → Provider → 실제 인프라 |
| HCL 기본 구조 | `provider { }`(무엇을 다룰지) + `resource "타입" "이름" { }`(무엇을 만들지) |
| 핵심 명령어 | `init`(Provider 준비) → `plan`(변경 예정 확인) → `apply`(실제 적용) → `destroy`(제거) |
| State 파일 | `terraform.tfstate` — Terraform이 "지금 무엇을 만들었는지" 기억하는 기록 |

### 10주차 — Terraform Cloud와 원격 State
| 개념 | 핵심 요약 |
|---|---|
| 로컬 State의 한계 | 협업 시 충돌 가능, 민감정보 평문 포함 위험, VM이 꺼지면 실행 불가 |
| Terraform Cloud | State를 원격 저장 + plan/apply를 원격 서버에서 실행(원격 실행) + 실행 이력(Runs) 기록 |
| 워크스페이스(Workspace) | 하나의 인프라 단위(State+변수+실행이력)를 관리하는 작업 공간 |
| **시험에 자주 나오는 함정** | 원격 실행은 Terraform Cloud **서버**에서 실행되므로 학생 로컬 Docker에 접근 불가 → `docker_container` 대신 `random_pet` 같은 로컬 비의존 리소스를 씀 |

### 11주차 — Jenkins CI/CD 서버 구축
| 개념 | 핵심 요약 |
|---|---|
| CI/CD | CI(코드 변경마다 자동 빌드·테스트) + CD(그 결과물을 자동 배포) |
| Jenkins 구성요소 | Jenkins 서버, Job(빌드 설정 단위), Build(실행 결과), Plugin(기능 확장) |
| Docker Compose 설치 | `jenkins_home` 볼륨(설정 영속성) + `/var/run/docker.sock` 마운트(호스트 Docker 제어) |
| Freestyle Job | 화면 클릭으로 Build Steps(Execute shell 등)를 설정하는 가장 단순한 Job 형태 |

### 12주차 — Jenkinsfile과 Poll SCM
| 개념 | 핵심 요약 |
|---|---|
| Jenkinsfile | 빌드 파이프라인을 코드로 선언 — Git으로 버전 관리·재사용 가능 |
| Declarative Pipeline 구조 | `pipeline { agent any  stages { stage("이름") { steps { ... } } } }` |
| withCredentials | 비밀번호 등 민감정보를 코드에 노출하지 않고 안전하게 사용 |
| **Webhook vs Poll SCM** | Webhook = GitHub이 Jenkins에 직접 알림(공인 접속 필요) / Poll SCM = Jenkins가 주기적으로 GitHub에 직접 확인(NAT 환경에서 필요) |
| Poll SCM 설정 | Build Triggers → Poll SCM 체크 → Schedule에 `* * * * *`(매 1분) |

## 핵심 명령어·코드 모음 (서술형·코드 트러블슈팅 대비)

| 구분 | 명령어/코드 | 용도 |
|---|---|---|
| Terraform | `terraform init / plan / apply / destroy` | 인프라 코드를 실제로 반영·제거 |
| Terraform | `terraform login` | CLI와 Terraform Cloud 계정 연동 |
| Terraform | `resource "random_pet" "lab" { length = 2 }` | 로컬 자원에 의존하지 않는 원격 실행 검증용 리소스 |
| Jenkins | `sh "docker build -t {id}/app:${BUILD_NUMBER} ."` | Jenkinsfile 안에서 이미지 빌드 |
| Jenkins | `withCredentials([usernamePassword(...)]) { ... }` | Credential을 안전하게 사용 |
| Jenkins | Build Triggers → Poll SCM → `* * * * *` | 자동 빌드 트리거 설정 |
| K8s(13주차 연결) | `docker exec minikube $(docker exec minikube sh -c "ls .../kubectl") --kubeconfig=/etc/kubernetes/admin.conf ...` | Jenkins 컨테이너에서 minikube 내부 kubectl 실행(네트워크 직접 연결 불가 우회) |

## 2차시 — 13~14주 통합실습 마무리 체크포인트

1차시가 개념 복습이라면, 2차시는 강의가 아니라 **점검**입니다. 학생 각자의 13~14주차 파이프라인이
실제로 끝까지 동작하는지 함께 확인하고, 막힌 부분을 그 자리에서 해결합니다.

### 체크포인트 진행 순서 (60분)

| 구간 | 시간 | 내용 |
|---|---|---|
| 도입 | 5분 | 오늘의 목적(점검 및 완주) 안내 |
| 전체 아키텍처 재확인 | 10분 | Git→Jenkins→Docker Hub→K8s 흐름도를 함께 다시 그려봄 |
| 자가 점검 체크리스트 수행 | 25분 | 아래 체크리스트를 각자 VM에서 직접 확인 |
| 트러블슈팅 라운드 | 15분 | 막힌 학생 위주로 순회 지도, 자주 나온 문제 전체 공유 |
| 마무리 | 5분 | 기말고사 대비 안내 + 질문 |

### 13~14주차 완주 체크리스트

- [ ] GitHub 저장소에 챗봇 코드(`app.py`, `Dockerfile`, `deployment.yaml`)가 올라가 있다
- [ ] Jenkins Pipeline Job이 Checkout → Build → Push → Deploy 4단계로 구성되어 있다
- [ ] `dockerhub-cred`가 Jenkins Credentials에 등록되어 있다
- [ ] Job의 Build Triggers에 Poll SCM(`* * * * *`)이 설정되어 있다
- [ ] 코드를 수정하고 `git push`하면 1분 내 자동으로 빌드가 시작된다
- [ ] 빌드 성공 후 `kubectl get pods`에서 chatbot Pod가 새 이미지로 교체된 것이 보인다
- [ ] `minikube addons enable metrics-server`가 활성화되어 있다
- [ ] chatbot Deployment에 resource request(낮은 값)가 설정되어 있다
- [ ] `kubectl get hpa`로 HPA가 생성되어 있는 것이 확인된다
- [ ] `ab` 부하 테스트로 HPA의 REPLICAS가 실제로 늘어나는 것을 확인했다
- [ ] `kubectl delete pod`로 Pod를 삭제해도 자동으로 새 Pod가 생성되는 것을 확인했다

### 자주 걸리는 지점과 해결 순서 (트러블슈팅 가이드)

| 걸리는 지점 | 확인 순서 |
|---|---|
| 자동 빌드가 안 됨 | ① Poll SCM 스케줄 문자열 정확한지 ② git push가 실제 반영됐는지(`git log`) ③ Jenkinsfile의 저장소 URL이 맞는지 |
| Deploy 단계 실패 | ① minikube 실행 중인지(`minikube status`) ② Deployment/컨테이너 이름이 YAML과 일치하는지 ③ docker exec 명령의 kubectl 경로가 정상 탐색되는지 |
| HPA가 안 움직임 | ① metrics-server 활성화 여부 ② resource request가 낮게 설정됐는지 ③ 챗봇 응답 로직에 실제 연산이 있는지 |
| 자가치유가 안 됨 | ① Pod가 아니라 Deployment로 배포됐는지(단순 Pod는 자동 재생성 안 됨) |

**(강의자 지시사항)** 이 체크리스트를 화면에 띄우고, 학생들에게 각자 VM에서 하나씩 확인하도록
안내하세요. 못 채운 항목이 있다면 그 항목의 원인을 위 표에서 찾아 그 자리에서 해결을 시도하고,
시간 안에 못 끝내면 LMS Q&A로 이어서 지원하겠다고 안내하세요.

## 자주 헷갈리는 포인트 (실제 강의 중 학생 질문 기반)

- Terraform Cloud의 원격 실행은 **서버**에서 이뤄지므로 학생 로컬 Docker에 접근 불가 — `random_pet`처럼 로컬에 의존하지 않는 리소스로 검증해야 한다.
- Webhook은 GitHub이 알려주는 방식, Poll SCM은 Jenkins가 스스로 물어보는 방식이다 — NAT 환경에서는 Poll SCM이 필요하다.
- Jenkins 컨테이너에서 kubectl을 바로 실행하면 minikube에 접속되지 않는다 — `docker exec`로 minikube 컨테이너 내부의 kubectl을 실행해야 한다.
- HPA가 동작하려면 metrics-server(지표 수집)와 낮은 resource request(임계값 계산 기준)가 함께 필요하다.
- 자가치유는 Pod 단위가 아니라 **Deployment가 관리하는** Pod에만 적용된다 — 단순 Pod는 삭제되면 재생성되지 않는다.
