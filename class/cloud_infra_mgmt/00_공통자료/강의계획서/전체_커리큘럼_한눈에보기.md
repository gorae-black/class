## 3. 주차별 상세

### 0주차 — VM 환경 구성 (선행, 필수)
| 구분 | 내용 |
|---|---|
| 목적 | 클라우드/리눅스 입문자가 설치 장벽 없이 바로 실습을 시작하도록, 교수자가 Docker·minikube·kubectl까지 전부 설치·초기화한 VM을 통째로 배포 |
| 진행 | VMware Workstation Player 설치 → zip 압축 해제 → cloud-lab.vmx 실행 → student/a1234 로그인 → 환경 검증 |
| 검증 | `docker --version` / `kubectl version --client` / `minikube status` |

### 1~3주차 — Docker 컨테이너 기초
| 주차 | 주제 | 1차시 | 2차시 | 3차시 |
|---|---|---|---|---|
| 1주차 | 클라우드 패러다임과 Docker 입문 | 클라우드·가상화(Hypervisor) 개념 | 컨테이너 vs VM + Docker 아키텍처 | Docker 데몬 원리 + nginx 실행·inspect |
| 2주차 | Docker 컨테이너 생명주기·데이터 보존 | 컨테이너 명령어 심화(run/ps/stop/rm/logs/exec) | Bind Mount 개념·실습 | Named Volume vs Bind Mount 비교 |
| 3주차 | Dockerfile·이미지 빌드·배포 | Dockerfile 문법(FROM/RUN/COPY/CMD/EXPOSE) | GitHub·Docker Hub 계정 생성 + Flask 앱 빌드 | 이미지 최적화(레이어 캐싱) + Docker Hub push(선택) |

- Killercoda(수업): `docker run hello-world` 등 빠른 체험 / VM(과제): 볼륨 바인딩·Flask 앱 컨테이너화 등 데이터 영속성 중심 실습
- 3주차 필수 제출물은 로컬 build+run+curl 확인까지이며, Docker Hub push는 선택 심화 과제

### 4~7주차 — Kubernetes 오케스트레이션
| 주차 | 주제 | 1차시 | 2차시 | 3차시 |
|---|---|---|---|---|
| 4주차 | K8s 아키텍처·Pod/Deployment | K8s 아키텍처 개관(Control Plane vs Node) | Pod 개념 및 첫 Pod 실습 | Deployment(ReplicaSet) 개념 + YAML 작성·배포 |
| 5주차 | Service와 내부 통신 | Service 개념(Pod IP 불안정성) | ClusterIP/NodePort/LoadBalancer 비교 + NodePort 실습 | Endpoints 확인 + 라운드로빈 부하분산 |
| 6주차 | ConfigMap/Secret·PV/PVC | ConfigMap과 Secret | PV/PVC 개념 + hostPath 실습 | MySQL 배포 + 재시작 후 데이터 영속성 검증 |
| 7주차 | Ingress·자가치유 | Ingress 개념(L7 라우팅) | Ingress 라우팅 규칙 실습(도메인 매핑) | 자가치유(Self-Healing) 실습 + 4~7주 정리 |

- Killercoda(수업): 멀티노드 클러스터 체험 / VM(과제): Minikube 로컬 K8s, PV/PVC, 자가치유 등

### 8주차 — 중간고사 (30%)
| 차시 | 내용 |
|---|---|
| 1차시 | 1~3주 복습(클라우드 개념·Docker 핵심) + 온라인 퀴즈 |
| 2차시 | 4~7주 복습(K8s 아키텍처·핵심 오브젝트) + 예상 문제 풀이 |
| 3차시 | ★ 온라인 시험 (이론 + kubectl 명령어 서술형 + YAML 오류 수정) |

### 9~10주차 — Terraform IaC
| 주차 | 주제 | 1차시 | 2차시 | 3차시 |
|---|---|---|---|---|
| 9주차 | Terraform 입문·HCL 문법 | IaC 개념 + Terraform 아키텍처 | Terraform 설치 + HCL 문법 + init/plan/apply | destroy + State 파일 개념(10주 예고) |
| 10주차 | Terraform Cloud·원격 State | 원격 State 필요성 + Terraform Cloud 개념 | Terraform Cloud 계정·워크스페이스 생성(가입 가이드) | GitHub 연동 + 원격 실행 확인 |

- 10주차에서 Terraform Cloud 계정이 처음 필요(3번째 외부 서비스 가입). 예제는 `random_pet` 사용(원격 실행은 학생 로컬 Docker에 접근 불가 — 검증 완료)

### 11~12주차 — Jenkins CI/CD
| 주차 | 주제 | 1차시 | 2차시 | 3차시 |
|---|---|---|---|---|
| 11주차 | Jenkins 서버 구축·Job 설계 | CI/CD 개념 + Jenkins 아키텍처 | Docker Compose로 Jenkins 설치 | 플러그인·관리자 계정 구성 + 첫 Job |
| 12주차 | Jenkinsfile·자동 빌드 연동 | Declarative Pipeline 문법 | Jenkinsfile 작성 + 수동 빌드 실습 | GitHub 연동(Poll SCM) + 자동 빌드 확인 |

- 학생 VM은 NAT 뒤에 있어 GitHub Webhook 직접 수신 불가 → **Poll SCM**(1분 주기) 방식 사용(검증 완료)

### 13~14주차 — 통합 실습 (100% 로컬 VM 전용)
| 주차 | 주제 | 1차시 | 2차시 | 3차시 |
|---|---|---|---|---|
| 13주차 | AI 챗봇 전체 자동배포 파이프라인 | 파이프라인 아키텍처 설계 | Git→Jenkins(Poll SCM)→Docker Hub→K8s 구축 | End-to-End 배포 테스트·트러블슈팅 |
| 14주차 | HPA 오토스케일링·자가치유 운영 | HPA 개념 + 오토스케일링 설정 | ApacheBench 부하 테스트 | 자가치유 실습 + 전체 과정 회고 |

- 구현 아키텍처: 학생 PC 소스 수정 → GitHub Push → Jenkins 감지(Poll SCM) → Docker 이미지 빌드·Push → K8s Deployment 자동 교체
- Jenkins→minikube 연결은 `docker exec minikube kubectl --kubeconfig=/etc/kubernetes/admin.conf ...` 방식 사용(네트워크 방식 대신 — 검증 완료)
- HPA 실습은 CPU request를 50m 이하로 낮추고 챗봇 응답 로직에 약간의 연산을 넣어야 스케일업이 관찰됨(검증 완료)

### 15주차 — 기말고사 (30%)
| 차시 | 내용 |
|---|---|
| 1차시 | 9~12주 복습(Terraform·Jenkins 핵심) + 온라인 퀴즈(코드 빈칸) |
| 2차시 | 13~14주 통합실습 마무리 체크포인트(트러블슈팅·완주 여부 점검) |
| 3차시 | ★ 온라인 시험(전체 파이프라인 아키텍처 분석·설계 서술형 + 코드 트러블슈팅) |
