## 2. 실습 체크리스트

### 1차시 — Ingress 개념 (★ 필수)
- [ ] `minikube addons enable ingress` 실행
- [ ] `kubectl get pods -n ingress-nginx` → STATUS Running 확인 — 캡처 첨부

### 2차시 — Ingress 라우팅 실습 (★ 필수)
- [ ] `echo "$(minikube ip) myapp.local" | sudo tee -a /etc/hosts` 실행
- [ ] `myapp.local`을 host로 지정하는 `ingress.yaml` 작성 및 적용
- [ ] `curl http://myapp.local` 접속 성공 화면 — 캡처 첨부

### 3차시 — 자가치유 실습 + 4~7주 정리 (★ 필수, 4~7주차 종합)
- [ ] `kubectl get pods -w &`로 실시간 모니터링 시작
- [ ] `kubectl delete pod {pod-name}` 실행 후, 자동 복구 과정(Terminating → 재생성 → Running) — 캡처 첨부
- [ ] 4주차(Pod/Deployment)~7주차(Ingress/자가치유) 실습 결과를 종합 정리

## 3. 자가 점검

- [ ] Ingress가 Service와 어떻게 다른지 설명할 수 있다
- [ ] 자가치유가 어떤 원리로 동작하는지("원하는 상태" 선언 + 지속 비교) 설명할 수 있다
- [ ] minikube 재시작 직후라면 Ingress 컨트롤러 준비까지 10~20초 대기 후 진행했다
