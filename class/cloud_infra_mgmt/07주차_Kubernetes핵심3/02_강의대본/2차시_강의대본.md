## [슬라이드 05] Ingress YAML의 핵심 구조

"Ingress YAML의 핵심은 `rules` 항목입니다. 이 아래에 `host`(도메인)와 `path`(경로)별로 라우팅 규칙을
나열합니다.

각 규칙은 `backend`에 연결할 Service 이름과 포트를 지정합니다. 예를 들어 `host`가 `myapp.local`이면
`myapp-service:80`으로 연결한다는 식이에요.

**(5주차 연결)** 여기서 backend에 지정하는 게 바로 5주차에서 배운 Service입니다. Ingress는 직접 Pod에
연결하지 않고, 항상 Service를 거쳐서 연결됩니다."

## [슬라이드 07] hosts 파일이 하는 일

"원래 우리가 아는 도메인 주소는, DNS 서버가 실제 IP 주소로 변환해줘야 접속이 됩니다.

그런데 hosts 파일에 '도메인 - IP' 짝을 직접 적어두면, DNS 조회 없이 그 IP로 바로 연결됩니다. 오늘
실습에서는 `myapp.local`이라는 가짜 도메인을 minikube의 IP에 매핑해볼 거예요.

화면에 보이는 것처럼 `echo \"$(minikube ip) myapp.local\" | sudo tee -a /etc/hosts` 명령으로
매핑하고, `curl http://myapp.local`로 접속을 확인합니다."

---

## [슬라이드 08] 이 실습이 실제 서비스와 다른 점

"한 가지 짚고 넘어갈 게 있는데요, 실제 서비스는 공인 DNS 서버에 도메인을 등록해서 전 세계 어디서든
접속할 수 있게 합니다.

**(중요한 구분)** 하지만 오늘 우리는 내 VM의 hosts 파일만 수정하는 거라서, 이 도메인은 **내 VM
안에서만** 동작합니다. 다른 컴퓨터에서는 `myapp.local`을 쳐도 연결되지 않아요.

그래도 Ingress가 도메인을 보고 라우팅을 처리하는 원리 자체는 완전히 동일합니다. hosts 매핑은 그
원리를 배우기 위한 '로컬 실습용 흉내'라고 생각하시면 됩니다."

---

## [슬라이드 09] Killercoda — 경로 기반 라우팅 실습

"자, 직접 해보겠습니다.

`/ko`, `/en` 경로로 서로 다른 컨테이너에 연결되는 Ingress YAML을 작성하세요. `kubectl apply`로
적용한 다음, 각 경로로 curl 요청을 보내서 서로 다른 응답이 오는지 확인해보세요.

**(관찰 포인트)** path 규칙이 host 규칙과 함께 어떻게 매칭되는지 잘 관찰해보세요. 같은 도메인이라도
경로에 따라 다른 곳으로 연결되는 걸 직접 눈으로 확인하는 게 오늘 실습의 핵심입니다.

**(화면 공유해 실제로 시연)** 제가 먼저 보여드릴게요."

## [슬라이드 10] VM 안내 — hosts 매핑+도메인 접속

"개인 VM에서 다음을 진행하세요.

첫째, `echo \"$(minikube ip) myapp.local\" | sudo tee -a /etc/hosts` 명령으로 도메인을 매핑하세요.
**(sudo 안내)** 이때 VM 계정 비밀번호를 입력해야 합니다.

둘째, `myapp.local`을 host로 지정하는 ingress.yaml을 작성하고 적용하세요.

셋째, `curl http://myapp.local`로 접속에 성공한 화면을 캡처해서 제출하시면 됩니다."

## 강의자 참고 — 예상 질문 & 답변

| 예상 질문 | 답변 가이드 |
|---|---|
| "curl http://myapp.local이 안 돼요" | 대부분 세 가지 원인입니다. ① hosts 파일에 매핑이 제대로 저장됐는지(`cat /etc/hosts`로 확인), ② Ingress 컨트롤러 Pod가 Running 상태인지, ③ ingress.yaml의 host 값과 curl 주소가 정확히 일치하는지 확인해보세요. |
| "hosts 파일을 잘못 건드리면 문제가 생기나요?" | 맨 아래에 한 줄만 추가하는 거라 기존 설정에는 영향이 없습니다. 다만 실습이 끝난 후 불필요해지면 그 줄만 지워도 무방합니다. |
| "실제 회사에서는 DNS를 어떻게 설정하나요?" | 도메인 등록기관에서 도메인을 구매하고, 그 도메인의 DNS 레코드가 클러스터의 공인 IP(보통 Ingress 컨트롤러의 LoadBalancer IP)를 가리키도록 설정합니다. 오늘 배운 hosts 매핑은 이 과정을 개인 VM 안에서 흉내낸 것입니다. |
