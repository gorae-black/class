## [슬라이드 05] 클라우드·리눅스가 처음이어도 괜찮습니다

**(핵심 강조)** 여러분이 할 일은 딱 두 가지입니다. VM 파일을 받아서 열고, student 계정으로

## [슬라이드 09] 설치 순서 5단계

"설치 순서는 다섯 단계입니다.

① VMware Workstation Player를 설치합니다.
② LMS에서 받은 cloud-lab-vm.zip을 원하는 위치에 압축 해제합니다. 그러면 cloud-lab.vmx와
cloud-lab.vmdk 두 파일이 담긴 폴더가 생깁니다.
③ VMware Player를 실행하고 'Open a Virtual Machine'을 클릭한 뒤, 압축 해제한 폴더 안의
cloud-lab.vmx 파일을 선택합니다.
④ 'Play virtual machine'을 클릭합니다. 잠시 후 검은 화면에 로그인 프롬프트가 뜹니다.
⑤ student 계정으로 로그인합니다.

**(같이 진행)** 자, 그럼 지금부터 저와 함께 순서대로 진행해보겠습니다. 먼저 zip 파일을 압축
해제해주세요."

## [슬라이드 13] 학기 중 이런 계정을 만들게 됩니다

"환경 확인은 끝났고, 마지막으로 학기 전체를 미리 그려보는 시간을 갖겠습니다. 지금 당장 만들 건
아니고, 이런 게 있다는 것만 미리 알아두시라는 취지입니다.

**GitHub** — 3주차 2차시에 가입합니다. 소스 코드를 저장하는 용도로, 9주차부터 13주차까지 계속
재사용됩니다.

**Docker Hub** — 마찬가지로 3주차 2차시에 가입합니다. 만든 이미지를 올리고 받는 용도로, 13주차
Jenkins 파이프라인에서도 그대로 재사용됩니다.

**Terraform Cloud** — 10주차 2차시에 가입합니다. 원격으로 인프라 상태를 관리하는 용도인데, GitHub
계정으로 가입하면 이후 연동이 더 편합니다.

**(안심시키기)** 세 서비스를 동일한 이메일 주소로 가입해두시면 나중에 관리가 편합니다.

**(Killercoda는 예외라는 점 강조)** 그리고 오늘부터 매 수업 시간에 쓰게 될 Killercoda는 **가입이
필요 없습니다.** 매 차시 제공되는 링크만 클릭하면 브라우저에서 바로 실습 환경이 뜹니다. 계정
걱정은 GitHub·Docker Hub·Terraform Cloud, 이 세 가지만 기억해두시면 됩니다."

## [슬라이드 14] 자주 발생하는 문제와 해결

"혹시 문제가 생겼다면 이렇게 해결해보세요.

**VM이 켜지지 않는다** → BIOS에서 가상화 옵션, 즉 Intel VT-x나 AMD-V가 켜져 있는지 확인해주세요.
이건 보통 노트북 제조사마다 BIOS 진입 방법이 달라서, 필요하면 개별적으로 도와드리겠습니다.

**로그인이 안 된다** → 아이디와 비밀번호는 student / a1234로 통일되어 있습니다. 대소문자를
정확히 확인해주세요.

**그 외 문제** → 지금 해결이 안 되면 LMS Q&A 게시판에 화면 캡처와 함께 남겨주시면, 제가 확인하고
답변드리겠습니다.

**(질의응답 시간)** 지금부터 질문 있으신 분들은 편하게 말씀해주세요."

## 강의자 참고 — 예상 질문 & 답변

| 예상 질문 | 답변 가이드 |
|---|---|
| "Mac이나 리눅스 PC를 쓰는데 VMware가 다른가요?" | VMware Fusion(Mac)이나 Workstation Player(리눅스)로도 동일한 vmx/vmdk 파일을 열 수 있습니다. 다만 이 강좌의 가이드는 Windows 기준으로 작성되어 있으므로, Mac/리눅스 사용 학생은 개별적으로 안내가 필요할 수 있습니다. |
| "VM 성능이 너무 느려요" | 호스트 PC의 다른 프로그램을 종료해 RAM 여유 공간을 확보해보세요. 그래도 느리다면 VMware 설정에서 VM에 할당된 CPU/RAM 수치를 확인해주세요(기본 2 vCPU, RAM 4GB로 설정되어 있습니다). |
| "VM을 끄면 실습했던 내용이 사라지나요?" | 아니요. VM은 일반 컴퓨터처럼 종료해도 디스크에 저장된 내용(설치한 것, 만든 파일 등)이 그대로 유지됩니다. VMware Player에서 정상적으로 Shut Down 하면 됩니다. |
| "노트북을 바꾸면 VM을 어떻게 옮기나요?" | cloud-lab-vm 폴더 전체(vmx, vmdk 파일 포함)를 그대로 복사해서 새 컴퓨터에 옮기고, 그 컴퓨터의 VMware Player에서 동일하게 열면 됩니다. |
| "계정을 지금 미리 만들어둬도 되나요?" | 네, 미리 만들어두셔도 상관없습니다. 다만 3주차·10주차에 각 서비스별로 단계별 가입 가이드를 다시 제공하니, 오늘은 어떤 계정이 언제 필요한지 정도만 기억해두셔도 충분합니다. |
| "Killercoda도 나중에 계정이 필요해지나요?" | 아니요. 이 과목에서 쓰는 Killercoda 시나리오는 전부 링크 클릭만으로 접속하는 방식이라 학기 내내 계정이 필요 없습니다. |
