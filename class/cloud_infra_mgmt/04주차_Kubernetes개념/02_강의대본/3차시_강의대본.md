## [슬라이드 05] 선언하면 K8s가 유지해준다

"Deployment의 핵심은 이겁니다 — '지금 nginx Pod 2개를 만들어라'라는 **명령**이 아니라, '항상 nginx
Pod가 2개 떠 있어야 한다'라는 **선언**을 하는 거예요.

Deployment는 내부적으로 ReplicaSet이라는 걸 만들어서 이 개수를 계속 감시합니다. 만약 Pod 하나가
어떤 이유로든 죽으면, ReplicaSet이 즉시 알아채고 새 Pod를 하나 더 만들어서 다시 2개를 채웁니다.

**(핵심 강조, 천천히)** '명령'이 아니라 '선언'이라는 이 차이가 K8s 전체를 관통하는 핵심 철학입니다.
우리는 원하는 '최종 상태'만 말해주고, 그 상태를 유지하는 건 K8s가 알아서 합니다."

## [슬라이드 07] YAML 기본 구조

"K8s에 무언가를 선언할 때는 YAML이라는 형식의 파일을 씁니다. 기본 구조는 네 부분입니다.

`apiVersion`은 어떤 버전의 K8s API를 쓸지 정하는 거고, Deployment는 `apps/v1`을 씁니다.
`kind`는 무엇을 만들지 — Deployment, Pod, 나중에 배울 Service 등을 적습니다.
`metadata`는 이름표 같은 부가 정보고, `spec`이 실제로 원하는 상태를 적는 핵심 부분입니다.

**(공감 유도)** 처음엔 낯설어 보이지만, 패턴이 항상 똑같아서 몇 번 써보면 금방 익숙해집니다."

---

## [슬라이드 08] deployment.yaml 예시

"실제 예시를 보겠습니다. `apiVersion: apps/v1`, `kind: Deployment`, `metadata`에 이름을 `myapp`으로
적고, `spec`에서 `replicas: 2`로 '2개 유지'를 선언합니다. 그리고 그 안에 어떤 이미지를 쓸지
`containers` 부분에 `nginx`를 적습니다.

**(들여쓰기 강조)** YAML에서 제일 중요한 건 들여쓰기입니다. 스페이스 칸 수가 안 맞으면 오류가 나니,
오늘 실습에서는 제공되는 템플릿을 그대로 복사해서 이름이나 replicas 숫자만 바꿔보는 것부터
시작하세요."

## [슬라이드 09] Killercoda — Deployment 선언 실행

"자, 직접 해보겠습니다. 먼저 `kubectl get nodes`로 지난 시간 복습을 한 번 하고요.

`deployment.yaml` 파일을 작성하세요 — replicas는 2, 이미지는 nginx로요. `kubectl apply -f
deployment.yaml`로 적용하고, `kubectl get pods -w`로 Pod 2개가 만들어지는 과정을 실시간으로
지켜보세요.

**(화면 공유해 실제로 시연)** 제가 먼저 보여드릴게요."

## [슬라이드 10] VM 과제 — minikube에 배포 + 자가치유 맛보기

"개인 VM에서 `minikube start` 후 같은 YAML로 `kubectl apply`, `kubectl get pods`로 2개가 Running인지
확인하세요.

**(재미있는 미리보기)** 여기서 하나만 더 해볼게요. `kubectl delete pod {파드 이름 중 하나}`로 Pod
하나를 강제로 지워보세요. 그리고 `kubectl get pods`를 다시 쳐보시면 — 신기하게도 다시 2개로 돌아와
있을 겁니다! 이게 아까 말씀드린 '선언적 관리'가 실제로 작동하는 모습이에요. 이 원리는 7주차
'자가치유' 시간에 자세히 다룰 거예요, 오늘은 맛보기입니다."

"오늘 배운 내용, 두 가지로 정리하겠습니다.

## 강의자 참고 — 예상 질문 & 답변

| 예상 질문 | 답변 가이드 |
|---|---|
| "YAML 문법이 너무 헷갈려요" | 정상입니다. 처음엔 제공된 템플릿을 그대로 복사해서 숫자·이름만 바꾸는 것부터 시작하고, 직접 처음부터 작성하는 건 나중 주차에서 점진적으로 늘려가면 됩니다(3단계 실습 구조: 따라하기→수정하기→스스로 작성). |
| "replicas를 5로 바꾸면 어떻게 되나요?" | kubectl apply를 다시 실행하면 K8s가 현재 상태(2개)와 원하는 상태(5개)를 비교해서 부족한 3개를 추가로 만들어줍니다. 이것도 선언적 관리의 예시로 언급해도 좋습니다. |
| "ReplicaSet을 직접 만들어야 하나요?" | 아니요, Deployment를 만들면 K8s가 ReplicaSet을 자동으로 만들어 관리해줍니다. 우리는 Deployment까지만 다루면 충분합니다. |
