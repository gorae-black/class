## 2. 실습 체크리스트

### 1차시 — K8s 아키텍처 개관 (자가 확인, 제출 선택)
- [ ] `minikube start` 실행
- [ ] `kubectl get nodes` → 노드 상태 Ready 확인

### 2차시 — Pod 개념 (★ 필수)
- [ ] `kubectl run mypod --image=nginx` 실행
- [ ] `kubectl get pods` 결과 — 캡처 첨부
- [ ] `kubectl describe pod mypod` 결과 — 캡처 첨부
- [ ] `kubectl logs mypod` 결과 — 캡처 첨부

### 3차시 — Deployment·자가치유 맛보기 (★ 필수)
- [ ] `deployment.yaml` 작성 후 `kubectl apply -f deployment.yaml`
- [ ] `kubectl get pods` → Pod 2개 Running 확인 — 캡처 첨부
- [ ] `kubectl delete pod {파드이름}` 실행 후 `kubectl get pods` 재확인 → 자동 복구(다시 2개) 확인 — 캡처 첨부

## 3. 자가 점검

- [ ] Pod와 Deployment의 차이를 한 문장으로 설명할 수 있다
- [ ] Pod를 강제로 삭제했는데 왜 다시 생겼는지 설명할 수 있다 ("선언적 관리" 개념)

## 4. 특이사항 / 질문 (선택)

> `kubectl apply` 직후 바로 `kubectl get pods`를 치면 "No resources found"가 나올 수 있습니다.
> 몇 초 기다리거나 `kubectl get pods -w`로 지켜보세요.
